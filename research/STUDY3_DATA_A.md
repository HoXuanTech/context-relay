## Summary1 — T1 Test Context (after Part 1 compression)
*Input: Part 1 (~6,100 words) → compressed to ~1,000 words*

This survey, authored by Lingfei Wu, Yu Chen, and colleagues from institutions including Pinterest, Rensselaer Polytechnic Institute, and JD.COM, presents a comprehensive overview of Graph Neural Networks applied to Natural Language Processing. The paper's central motivation is that while deep learning on sequential data has dominated NLP, many language problems are naturally expressed as graphs — dependency trees, AMR graphs, knowledge graphs, information extraction graphs — and standard RNN/CNN architectures cannot directly exploit these structures. The survey proposes a taxonomy organizing GNN-for-NLP research along three axes: graph construction, graph representation learning, and graph-based encoder-decoder models, with a fourth axis covering applications.

Natural language can be viewed through three representational lenses: as a bag of tokens (ignoring order), as a sequence of tokens (the dominant modern view), or as a graph (capturing relational structure among elements). Traditional graph-based NLP methods — random walk algorithms, graph clustering, graph matching, and label propagation — were applied to tasks like word-sense disambiguation, text clustering, and sentiment analysis. However, these methods share key limitations: limited expressive power, no unified learning framework, and an inability to jointly model both graph structure and node/edge features. GNNs emerged as a unified message-passing framework that overcomes these limitations by transforming and aggregating node and edge features through neural layers.

The foundational GNN operation is graph filtering, where updated node embeddings at layer l are computed as a function of the adjacency matrix and the node embeddings from layer l−1. This does not alter graph structure but refines representations through stacked layers. For graph-level tasks, graph pooling is introduced: it takes the graph and node embeddings as input and produces a smaller graph with fewer nodes and new embeddings, ultimately collapsing to a single graph-level representation.

Four categories of graph filters are introduced. Spectral-based filters, exemplified by Graph Convolutional Networks (GCN, Kipf and Welling 2016), operate through eigendecomposition of the normalized graph Laplacian; the full computation is expensive, so a Chebyshev polynomial truncation approximates it, and GCN simplifies further with a layer-wise renormalization trick producing the well-known formula H(l) = σ(D̃^{-1/2} Ã D̃^{-1/2} H(l-1) W(l-1)). Spatial-based filters use message passing directly: MPNN (Gilmer et al. 2017) provides a general framework where nodes send messages along edges and update their states, while GraphSage (Hamilton et al. 2017) uses neighborhood sampling for scalability. Attention-based filters, represented by GAT (Velickovic et al. 2018), compute attention scores between a node and its neighbors via a shared weight vector and LeakyReLU, using multi-head attention for stability. Recurrent-based filters use GGNN (Gated Graph Neural Networks), which applies GRU units across fixed unrolling steps, with separate parameters for edge types and directions encoded in the matrix A.

Graph pooling is divided into flat and hierarchical variants. Flat pooling aggregates all node embeddings in a single step using max-pooling, average-pooling, or BiLSTM-based aggregation. Hierarchical pooling progressively coarsens the graph — either by sub-sampling important nodes or by merging nodes into supernodes — and has been shown to capture more complex structural patterns.

The survey then previews the problem of graph construction for NLP: since most NLP inputs are raw text sequences rather than graphs, transforming text into useful graph structures is a critical and non-trivial pipeline step. The paper organizes this into static construction (using parsing tools and fixed rules, typically done at preprocessing) and dynamic construction (learned jointly with the downstream task). This foundational material sets up the survey's coverage of eleven static graph types and several dynamic approaches described in later sections.

## Summary2 (after Part 2 compression)
*Input: Summary1 + Part 2 (~7,070 words) → compressed to ~1,000 words*

This survey covers GNNs for NLP along three main axes: graph construction, graph representation learning, and encoder-decoder models. The foundational GNN building blocks are graph filtering — which refines node embeddings through stacked message-passing layers without changing graph structure — and graph pooling, which reduces graph size to produce graph-level representations. Core filter architectures include GCN (spectral, renormalization-based), GraphSage (spatial, sampled neighborhoods), GAT (attention-weighted message passing), and GGNN (recurrent, GRU-gated, edge-direction-aware).

For graph construction, the paper distinguishes static and dynamic approaches. Static construction uses fixed preprocessing based on domain knowledge or parsing tools. Eleven categories of static graphs are described. Dependency graphs extract syntactic dependency relations from parsing trees (using tools like Stanford CoreNLP), supplemented with sequential edges to preserve word ordering; together they capture grammatical relationships at the word level. Constituency graphs capture phrase-level syntactic structure using non-terminal and terminal nodes from constituency parse trees, also augmented with sequential edges. AMR graphs are rooted, labeled, directed acyclic graphs encoding high-level semantic abstractions; sentences with similar meaning share the same AMR representation, making AMR more semantically general than syntactic parse structures. Information extraction graphs use coreference resolution and OpenIE-style triple extraction to build relational graphs of entities and their predicates across sentences; coreference clusters are collapsed into single nodes. Discourse graphs model logical ordering constraints between sentences, typically derived from RST parsing, and are useful for handling long-document dependencies. Knowledge graphs either encode document content as structured triples or incorporate external knowledge bases like ConceptNet; subgraph extraction via methods like Personalized PageRank filters noise. Coreference graphs explicitly link phrases referring to the same entity, rather than collapsing them. Similarity graphs compute node similarity (using TF-IDF, embeddings, cosine similarity) and apply sparsification via k-NN or threshold methods to reduce noise. Co-occurrence graphs connect words based on co-occurrence frequency within a sliding window, with edge weights computed by raw frequency or PMI. Topic graphs use LDA or similar methods to link documents to their latent topics. Application-driven graphs are task-specific designs — such as SQL parsing trees — that do not fit standard categories.

Many works combine multiple graph types into hybrid graphs by merging edge sets (if node sets are shared) or merging both node and edge sets (heterogeneous hybrid). This allows capturing complementary structural signals simultaneously.

Dynamic graph construction addresses the limitations of static approaches: their reliance on domain expertise, susceptibility to parsing errors, inability to correct construction errors during training, and task-agnostic nature. Dynamic methods jointly learn the adjacency matrix and the GNN representation end-to-end. The key components are similarity metric learning — which infers the weighted adjacency matrix from node embedding similarity — and sparsification, which extracts a sparse graph from the resulting fully connected one. Similarity metric functions include attention-based (dot-product variants with learnable parameters) and cosine-based (multi-head weighted cosine similarity) approaches. Structure-aware variants additionally incorporate existing edge information. Sparsification uses kNN-style top-k selection or epsilon-thresholding; regularization terms encouraging sparse graphs can also be used implicitly. A key finding is that combining intrinsic (static) and learned (implicit) graph structures improves performance, as the intrinsic structure provides a useful inductive prior and can stabilize training. Three learning paradigms are noted: end-to-end joint optimization, layer-adaptive graph learning (each GNN layer sees a different graph), and iterative refinement of graph structure and representations alternately.

## Summary3 — T2 Test Context (after Part 3 compression)

This survey organizes GNN-for-NLP research along three axes: graph construction, graph representation learning, and encoder-decoder models. Foundational GNN components are graph filtering (refining node embeddings through message passing without altering graph structure) and graph pooling (reducing graphs to produce graph-level representations). Core architectures include GCN, GAT, GraphSage, and GGNN. Static graph construction produces eleven graph types including dependency, constituency, AMR, IE, discourse, knowledge, coreference, similarity, co-occurrence, topic, and application-driven graphs; hybrid graphs combine multiple types. Dynamic graph construction jointly learns adjacency matrices with downstream tasks via similarity metric learning and sparsification, optionally combining learned and intrinsic graph structures.

Section 5 covers graph representation learning, organized by graph type: homogeneous, multi-relational, and heterogeneous graphs. For homogeneous graphs, heterogeneous source graphs are first converted by treating all edges as simple connectivity and discarding edge-type labels, then standard GNNs (GCN, GAT, GGNN, GraphSage) are applied. Directed graphs can be made undirected by averaging edge weights, or handled directly by models like GGNN and GAT. Dynamic graphs jointly learn structure and representation, with early works using RNN-like state encodings and later works using attention-based or metric-learning-based adjacency inference. Bidirectional GNN variants are described: BiGraphSage separately encodes forward and backward directions and concatenates; bidirectional GGNN fuses them via a gated function (Fuse using a learnable gate z) before a GRU update; a general bidirectional framework by Ribeiro et al. processes both directions with any GNN and concatenates with the original features, followed by a Bi-LSTM for final representation.

Multi-relational graphs (single node type, multiple edge types) require relation-aware message passing. R-GCN (Schlichtkrull et al. 2018) groups neighbor nodes by relation type and applies separate weight matrices, using basis decomposition or block-diagonal decomposition to prevent over-parameterization. Directed-GCN shares projection weights across directions but keeps relation-specific biases. R-GGNN (Beck et al. 2018) applies GRU-style recurrent updates with relation-specific weight matrices for reset and update gates. R-GAT has two variants: one adds relational heads computing attention over relation-grouped neighbors and fuses with standard attention; another learns global edge type representations as the absolute difference between mean source and target node embeddings. Gating mechanisms can be added to any multi-relational GNN to prevent over-smoothing by learning a gate controlling how much update information propagates.

Graph Transformers bridge GNNs and Transformers by incorporating structural information into self-attention. R-GAT-based graph transformers use GAT-style neighborhood aggregation per relation type and fuse results via feed-forward layers; bidirectional variants compute forward and backward aggregation and merge using a gated function. Structure-aware self-attention transformers retain the standard fully-connected attention but modify the attention score computation to include edge embeddings. Shaw et al. 2018 adds relative position encodings to the attention score. Xiao et al. and Zhu et al. incorporate edge embeddings in both the attention score and value computation. Cai and Lam extend this to bidirectional edge embeddings in the dot product. Edge embeddings can be computed from relative position tables, path-as-string lookups, or bidirectional GRU encoding of label sequences along shortest paths.

Heterogeneous graphs (multiple node and/or edge types) are handled via Levi Graph Transformation, which converts edges to new nodes, producing a bipartite graph with a single edge type but multiple node types. Meta-path-based methods (HAN) use type-specific transformations to project all nodes to a unified space, then perform node-level attention aggregation along each meta-path, followed by meta-path-level attention to weight different meta-paths. MEIRec is a heterogeneous GNN for recommendation that processes meta-path neighbors hop-by-hop recursively. R-GNN-based heterogeneous methods use type-specific aggregation without requiring manually defined meta-paths: HGAT performs two-level aggregation (type-level relevance, then node-level attention); MHGRN extends R-GCN to multi-hop path-based aggregation; HGT (Hu et al. 2020) is a graph transformer for heterogeneous graphs with meta-relation-specific attention, message passing, and projection, plus a relative temporal encoding strategy for time-dependent graphs.

## Summary4 (after Part 4 compression)
*Input: Summary3 + Part 4 (~7,700 words) → compressed to ~1,000 words*

This GNN-for-NLP survey covers graph construction (static: eleven types; dynamic: learned jointly with task), graph representation learning for homogeneous, multi-relational, and heterogeneous graphs, and encoder-decoder frameworks. Key representation models include GCN, GAT, GraphSage, GGNN, bidirectional variants, R-GCN, R-GGNN, R-GAT, graph transformers with structure-aware self-attention, HAN, HGT, and MHGRN. Levi graph transformation converts edge types to nodes for homogeneous GNN processing.

Section 6 covers GNN-based encoder-decoder models. Standard Seq2Seq models use RNN encoders and decoders, augmented with attention (soft alignment over encoder states), copying mechanisms (learnable token copying from input), coverage mechanisms (penalizing repeated attention), and scheduled sampling (mixing ground-truth and predicted inputs during training). These have achieved strong results in machine translation, summarization, and dialog, but cannot natively handle graph-structured inputs.

Graph-to-Sequence (Graph2Seq) models replace the sequence encoder with a GNN encoder while keeping sequence decoders. GNN encoders use GCN, GGNN, GraphSage, or GAT, extended to handle directed edges (separate parameters for incoming/outgoing/self-loop edges) and multi-relational graphs (relation-specific parameters or Levi graph conversion). Node and edge initialization typically uses BiRNNs applied to word embeddings; some works use BERT or RoBERTa embeddings with BiRNNs. Decoder techniques from Seq2Seq (attention, copy, coverage) carry over with modifications — notably, node-level copying replaces token-level copying. Some Graph2Seq systems fuse a sequential encoder and graph encoder before decoding; a few use reinforcement learning loss alongside cross-entropy for training. Transformer-based decoders are also used. The relationship between Graph2Seq and transformer-based Seq2Seq is noted as increasingly blurred since graph transformers combine GNN structure with attention architectures.

Graph-to-Tree (Graph2Tree) models address tasks where outputs are structured as trees — semantic parsing, math word problem solving, code generation. The graph construction stage typically uses syntactic dependency or constituency graphs, sometimes augmented with numerical or schema relations. Graph encoders use GCN, GGNN, or bidirectional GraphSage. The attention module distinguishes node types (sentence words vs. parse tree nodes) for better bridging. Tree decoders come in two forms: BFS-based decoders represent sub-trees as non-terminal nodes and branch recursively; DFS-based decoders treat generation as an action sequence progressing depth-first. Techniques like parent feeding, sibling feeding, and sub-tree copying enrich the tree decoder's contextual information.

Graph-to-Graph models handle graph transformation problems — converting an input graph to an output graph of different structure or semantics. Applications include information extraction (dependency/constituency graph → entity-relation graph) and semantic parsing (input graph → AMR/logic graph). In information extraction, challenges include differing node granularity (words vs. entities), multi-level context (word and sentence), and the dependency between co-reference links and entity relations. An end-to-end approach uses a hierarchical graph encoder (processing dependency, adjacent, and interactive edges across word-level and sentence-level nodes) and a parallel decoder that jointly predicts co-reference links and entity relations via deconvolution layers. Sequence-to-graph is a special case where the input sequence is treated as a line-graph; AMR parsing is formalized as two-stage node prediction followed by edge prediction using a pointer network.

Collectively, Graph2Seq handles graph input to sequence output, Graph2Tree adds structured tree output, and Graph2Graph handles fully structured input-output transformation. These encoder-decoder frameworks inherit challenges from both GNN representation learning (multi-relational modeling, over-smoothing, scalability) and sequence generation (exposure bias, cross-entropy limitations).

## Summary5 (after Part 5 compression)
*Input: Summary4 + Part 5 (~7,500 words) → compressed to ~1,000 words*

This survey organizes GNNs for NLP along graph construction (static: eleven types; dynamic: learned end-to-end), graph representation learning (homogeneous, multi-relational, heterogeneous), and encoder-decoder models (Graph2Seq, Graph2Tree, Graph2Graph). Core GNN architectures include GCN, GAT, GraphSage, GGNN, bidirectional variants, R-GCN, R-GGNN, R-GAT, graph transformers (structure-aware self-attention), HAN, HGT, and MHGRN. Encoder-decoder models replace sequence encoders with GNNs while retaining attention, copy, and coverage decoding techniques; tree decoders use BFS or DFS traversal strategies.

Section 7 surveys twelve NLP application areas. In neural machine translation, GNN-based approaches cast NMT as Graph2Seq, constructing syntactic dependency graphs, AMR graphs, SRL-based dependency graphs, lattice graphs (encoding segmentation ambiguity), relative position graphs, or multi-modal graphs combining text and images. Graph encoders include directed GCN, relational GGNN, structure-aware transformers with path-based edge embeddings, and graph transformers with cross-modal GAT fusion. Benchmarks use WMT datasets with BLEU evaluation.

In summarization, extractive and abstractive methods both exploit graph structure. Single-document approaches construct hybrid graphs (coreference + sequential), discourse graphs (RST), similarity graphs, or heterogeneous graphs with named entity, word, and sentence nodes. Multi-document approaches use discourse or topic graphs across document sets. Code summarization builds code graphs from ASTs or sub-token sequential and lexical links. GNN encoders include relational GGNN, GAT, and cross-attention between word and sentence sub-graphs. BERT and ALBERT are commonly used for node initialization. Benchmarks include CNN/DailyMail, NYT, WikiSum; metrics are ROUGE and BLEU.

Structural-data-to-text (AMR-to-text, SQL-to-text, RDF-to-text) converts explicit graph inputs to natural language. AMR graphs are converted to Levi graphs and processed by relational GGNN, heterogeneous GAT, multi-hop GCN, or structure-aware graph transformers. SQL graphs are built from sub-clauses and encoded by bidirectional GraphSage. RDF triples are treated as graphs with relation nodes. Benchmarks include LDC AMR releases and WikiSQL.

Question generation constructs dependency or SRL-based graphs (for QG from text), entity graphs with coreference links (for multi-hop QG), or KG subgraphs (for QG from KG). Dynamic graph construction via attention is also used. GCN, GAT, and GGNN serve as encoders; some models fuse sequence and graph encoders. Benchmarks include SQuAD and HotpotQA.

Machine reading comprehension builds entity graphs or hierarchical graphs (word/sentence/paragraph/document nodes) to support multi-hop, conversational, or numerical reasoning. Dynamic graph construction is used in conversational MRC. Multi-relational GNNs, especially gating RGCN variants, dominate. Pre-trained embeddings (GloVe, ELMo, BERT, RoBERTa) are standard for initialization. Benchmarks include HotpotQA, WikiHop, CoQA, and DROP.

Knowledge base QA extracts subgraphs from KGs and applies MHGRN or GAT with node-type and edge-type aware message passing to find answers. Open-domain QA fuses KG subgraphs with Wikipedia documents in heterogeneous graphs, updated via R-GCN and document aggregation iteratively. Community QA uses multi-modal occurrence graphs and APPNP-based hierarchical pooling.

Dialog systems use GNNs for state tracking (heterogeneous schema graphs), response generation (directed utterance graphs), and next utterance selection (topic-linked utterance graphs). Both static domain-knowledge graphs and dynamic Bernoulli-parameterized adjacency matrices have been explored.

Text classification builds corpus-level heterogeneous graphs (word and document nodes connected by co-occurrence and TF-IDF) for semi-supervised settings, or per-document word graphs for inductive settings. GCN, GGNN, and TensorGCN are applied; BERT, GloVe, or one-hot initializations are used. Benchmarks include 20NEWS, Ohsumed, AGNews.

Text matching uses word-lattice graphs (encoding segmentation alternatives) or concept interaction graphs (keyword co-occurrence clustered into concepts) with GCN or bidirectional GNN matching modules. Benchmarks are primarily Chinese datasets (LCQMC, BQ).

## Summary6 — T3 Test Context (after Part 6 compression)
*Input: Summary5 + Part 6 (~9,800 words) → compressed to ~1,000 words*

This survey covers GNNs for NLP across graph construction (static eleven-type taxonomy and dynamic end-to-end learned), graph representation learning (homogeneous, multi-relational, and heterogeneous GNNs including R-GCN, R-GAT, graph transformers, HAN, HGT), and encoder-decoder models (Graph2Seq, Graph2Tree, Graph2Graph). Twelve NLP application areas are surveyed including NMT, summarization, structural-data-to-text, question generation, MRC, KBQA, open-domain QA, community QA, dialog systems, text classification, text matching, and several more detailed below.

Topic modeling uses GCN-based autoencoders on biterm or document-word graphs; one approach interprets GAT as semi-amortized inference for stochastic block models. Benchmarks include 20NEWS and Reuters; topic coherence is the main metric.

Sentiment classification primarily uses dependency trees as input graphs, with GCN, GAT, and graph transformers as encoders; position-weighted embeddings near aspect terms are a common trick. R-GCN and R-GAT handle multi-relational cases; document-level methods use bipartite sentence-aspect graphs. Benchmarks include SemEval and Twitter datasets.

Knowledge graph tasks cover completion and alignment. KG completion is framed as link prediction using GNN encoders (R-GCN, attention-based GNNs) over multi-relational graphs combined with decoder scoring functions such as DistMult, ComplEx, ConvE, and TransE; training uses cross-entropy or margin loss with negative sampling. KG alignment (entity alignment across different KGs) uses GCN, GAT, and relation-aware methods; distance metrics include L1, L2, and cosine similarity. Benchmarks include FB15k-237, WN18RR for completion and DBP15K for alignment.

Information extraction covers NER and relation extraction. Dependency-based graphs, coreference graphs, and co-occurrence graphs are common constructions. GCN is the dominant model; attention-guided GCN prunes irrelevant dependency edges. Joint learning models (GraphRel, and others) couple NER and RE to reduce error propagation. Dynamic span graph construction selects confident entity spans on the fly. Benchmarks include ACE2005, TACRED, and SciERC.

Parsing covers dependency parsing (GNN as high-order encoder alongside BiLSTM, with dynamic edge weights), constituency parsing (GNN encoding partial trees in transition-based decoding), SQL parsing (GGNN over schema graphs, GNN-based reranking), and AMR parsing (fusing dependency and latent probabilistic graph representations). Benchmarks include PTB, UD, ATIS, WikiSQL, and AMR annotation releases.

Reasoning tasks include math word problem solving (Graph2Tree with GNN encoding number relationship graphs and BFS tree decoder), natural language inference (GNN-augmented with external knowledge graphs, subgraph filtering via PageRank to reduce noise), and commonsense reasoning (GNN retrieval from ConceptNet schema graphs). The Graph4NLP library (built on DGL and PyTorch) is highlighted as an open-source toolkit implementing the pipeline.

The final challenges and future directions identify five main areas. Dynamic graph construction remains limited: most work handles only homogeneous dynamic graphs, pair-wise similarity computation has O(n²) complexity limiting scalability to large KGs, and dynamic methods do not consistently outperform static ones. The GNN-vs-Transformer question notes that transformers are special-case GNNs operating on fully connected dynamic graphs; graph transformers combining structure-aware attention with GNN inductive biases represent the current frontier, but better approaches for multi-relational and heterogeneous graph inputs are still needed. Graph-to-graph models face challenges in end-to-end integration of sub-tasks, handling differing input/output graph concepts, graph sparsity, and unpaired supervision. Knowledge graph challenges include entity alignment ambiguity, large-scale subgraph extraction, and incomplete KG data. Multi-relational GNNs must balance over-parameterization against expressive power; the pre-training of GNNs for NLP, analogous to BERT-style pre-training for transformers, is identified as a promising but underexplored direction.
